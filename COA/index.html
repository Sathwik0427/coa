Core Topics in Computer Organization & Architecture

Basic Computer Structure

Computer organization refers to the operational units and their interconnections that realize the architectural specifications. It includes the detailed hardware design of the computer system.

Key Components:

  • CPU (Central Processing Unit): The brain of the computer that performs arithmetic and logical operations.
  • Memory Units: Storage areas for data and instructions (RAM, ROM, Cache).
  • Input/Output Devices: Interfaces that allow communication with the external world.
  • System Bus: Communication pathways between components.

Data Representation

Data representation involves how information is encoded and stored within a computer system. Understanding these concepts is fundamental to computer organization.

Number Systems:

  • Binary (Base-2): The fundamental number system used in computing (0s and 1s).
  • Hexadecimal (Base-16): Used as a human-friendly representation of binary values.
  • Octal (Base-8): Historical importance in early computing systems.

Data Formats:

  • Fixed-Point Representation: For integers and fixed decimal point numbers.
  • Floating-Point Representation: For representing real numbers with varying precision.
  • Character Encoding: Schemes like ASCII, Unicode for representing text.

Computer Architecture

Computer architecture defines the conceptual design and fundamental operational structure of a computer system. It serves as a blueprint for system designers.

Architecture Types:

  • Von Neumann Architecture: Uses a single memory for both data and instructions.
  • Harvard Architecture: Uses separate memories for data and instructions.
  • Modified Harvard Architecture: A hybrid approach used in modern systems.

Instruction Set Architecture (ISA):

  • CISC (Complex Instruction Set Computer): Many specialized instructions.
  • RISC (Reduced Instruction Set Computer): Fewer, optimized instructions.
  • MISC, VLIW, EPIC: Other architectural approaches.

The Von Neumann architecture uses a single memory space for both data and instructions, which can create a bottleneck known as the "Von Neumann bottleneck."

The Harvard architecture uses separate memory spaces for data and instructions, allowing for simultaneous access and potentially higher performance.

Modern processors often use a modified Harvard architecture with separate L1 caches for instructions and data, but unified L2/L3 caches and main memory.
